Key Scheduling Phases

üîç
Filtering
Predicate Phase - Remove Unsuitable Nodes

The filtering step finds the set of Nodes where it's feasible to schedule the Pod.

  • Sufficient CPU and memory resources
  • Pod resource requests can be met
  • Node selectors match pod requirements
  • Taints and tolerations align
  • Pod affinity/anti-affinity satisfied
  • Volume provisioning possible
Learn More ‚Üí
‚≠ê
Scoring
Priority Phase - Rank Remaining Nodes

The scoring step ranks the remaining nodes to choose the most suitable placement.

  • Least requested resources (resource balance)
  • Pod affinity preferences
  • Topology spread constraints
  • Image locality on nodes
  • Custom scoring plugins
Learn More ‚Üí
‚úì
Binding
Assignment Phase - Commit the Decision

The scheduler assigns the Pod to the highest-scoring Node.

  • Scheduler notifies API server
  • pod.spec.nodeName is updated
  • Binding persisted to etcd
  • Kubelet detects pod assignment
  • Container runtime creates container
  • Pod transitions to Running state
